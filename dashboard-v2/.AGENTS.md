# Dashboard-v2 Module: Visual Analytics

This directory contains the "Brass Vision" dashboard, a modern web application for analyzing RL agent behavior and game simulations.

## Contents & Behavior

- **`src/`**: React source code using TypeScript and Vite.
    - **`GameBoard.tsx`**: High-performance SVG-based rendering of the Brass map, including cities, slots, routes, and resource flows.
    - **`App.tsx`**: Main entry point; handles loading of evaluation indices and game traces.
    - **`StrategyHub.tsx`**: Heatmap visualization of model activity (e.g., build locations, move usage).
    - **`GameHUD.tsx` / `StepFooter.tsx`**: UI components for player state monitoring and "VCR" replay controls.
- **`public/data/`**: (Expected) Data directory where Python evaluation traces are stored.

## Key Features

- **VCR Replay**: Step-by-step navigation through game actions with full state visualization.
- **Diagnostics**: Displays Victory Point (VP) breakdowns, income, and resource consumption (Coal/Iron parasitism).
- **Strategy Analysis**: Aggregated heatmaps showing which cities and industries the winner built vs. the loser.
- **Responsive Layout**: Designed for high-fidelity technical analysis on large screens.

## Design Patterns

- **State-Driven Rendering**: The UI is a pure function of the "Engine State" JSON provided in the traces.
- **Glassmorphism / Neon Aesthetic**: Premium UI design for high engagement.
- **Framer Motion**: Smooth transitions between game steps and UI states.

## Key Linkages

- **Consumes**: JSON Traces generated by `python/test_agent.py`.
- **Relationship**: Provides a visual debugger for the `engine` logic and `python` agent policy.
