# Python Module: RL Training & Evaluation

This directory contains the Reinforcement Learning (RL) pipeline for Project Brass, including environment wrappers, model architectures, and training scripts.

## Contents & Behavior

- **`brass_env/`**: A Gym/Gymnasium wrapper that communicates with the Go `server` over HTTP. 
    - Handles high-speed tensor decoding (binary/base64).
    - Provides a standard interface for `stable-baselines3`.
- **`train.py`**: The main training script. Features the **Neural Architecture v2.0 (Expert Compatibility Matrix)**:
    - **Expert Extractor**: Separates Board (global) and Hand (local slot-based) observations.
    - **Expert Policy**: Implements a dot-product compatibility matrix between "Board Needs" and "Card Profiles" to select actions, optimizing for the large structured action space.
    - **Curriculum Learning**: Implemented via `CurriculumCallback`, which phases out dense rewards (VP/Income pulses) as the agent hits performance targets.
- **`test_agent.py`**: Evaluation script. Generates JSON "traces" including full game states, used by the dashboard for visualization.
- **`runs/`**: Contains TensorBoard logs, checkpoints, and final model weights.

## Neural Architecture: Expert Compatibility Matrix

The v2.0 architecture solves the structured action space (Card Slot x Action Type) by:
1. Encoding the **Board** to define "what we need" (Action Profiles).
2. Encoding each **Card** in hand to define "what we can do" (Card Profiles).
3. Using a **Batch Matrix Multiplication (BMM)** to find the highest compatibility between cards and needs.

## Key Linkages

- **Communicates With**: `server/` (Go API) to drive the game engine.
- **Outputs To**: `eval_data/` (traces/index) which is consumed by the `dashboard-v2` frontend.
- **Dependencies**: `stable-baselines3`, `sb3-contrib` (MaskablePPO), `torch`, `uv`.
